# MOODWAVE Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a localhost Next.js and FastMCP application that streams Qwen-curated playlists made only from externally verified tracks and saves them to SQLite only on explicit user action.

**Architecture:** Next.js owns input validation, the bounded Qwen tool loop, NDJSON streaming, and UI. Python FastMCP owns music providers, deterministic playlist composition, and SQLite; the web app never opens the database directly.

**Tech Stack:** Next.js App Router, TypeScript, React, Tailwind CSS, Zod, Vitest, NVIDIA OpenAI-compatible API, MCP TypeScript SDK, Python 3.13, FastMCP, Pydantic, HTTPX, sqlite3, pytest.

## Global Constraints

- Create the application under `moodwave/`; preserve the approved design in `docs/superpowers/specs/2026-07-20-moodwave-design.md`.
- Expose exactly eight MCP tools: four agent music tools and four explicit playlist storage tools.
- Use Qwen through NVIDIA Chat Completions with `tool_choice="auto"`, temperature `0.2`, and an eight-iteration ceiling.
- Never add a track that is absent from `verify_music_tracks` and `compose_playlist` output.
- Stream `status`, `tool`, `tool_result`, `text_delta`, `playlist`, `error`, and `done` as NDJSON; never expose private reasoning.
- SQLite is owned exclusively by FastMCP; enable foreign keys, bind parameters, and transact writes.
- Keep API secrets in environment variables and never print their values.
- Recommendation generation never persists data; only the save button calls `save_playlist`.
- Report mock/automated verification separately from live NVIDIA and external-provider verification.
